# Mutation Arena — Mobile

A **lightweight Android app** that runs two fine-tuned **Gemma** models **fully
on-device** to call buy/sell/hold signals on live markets. It's the product end
of the desktop *Mutation Arena*: the desktop evolves + fine-tunes the agents,
this app just bundles the trained models and shows the trading view.

> Kotlin · Jetpack Compose · MediaPipe LLM Inference (on-device Gemma) · no server.

## How it fits together

```
Desktop Mutation Arena                         Mutation Arena Mobile (this app)
──────────────────────                         ───────────────────────────────
evolve + RL-train agents                       bundles  scout.task + analyst.task
   │                                              │  (Gemma, fine-tuned)
mine winning decisions → corpus                runs them ON-DEVICE (MediaPipe)
   │                                              │
LoRA fine-tune Gemma ×2  ───────────────────►  live chart + 2 model signals
   (Scout, Analyst)        ship the .task files   (fully offline inference)
```

## What's here

- **One screen** (`ui/TradingScreen.kt`): market chips (BTC/ETH/SOL), a
  **scalping-first timeframe row (1m/5m/15m/30m/1h/4h)**, a Compose candlestick
  chart, and a **Run models** button that streams both models' signals.
- **Real data** (`data/KrakenRepository.kt`): OHLC from Kraken's public REST API.
  No fabricated candles — empty/error state on failure.
- **On-device LLM** (`model/GemmaModel.kt`, `model/SignalEngine.kt`): each Gemma
  `.task` is loaded via the MediaPipe **LLM Inference** API and prompted with the
  recent price action to return `{action, confidence, reason}`.
- Kraken-dark theme; green = buy/up, red = sell/down; purple reserved for brand.

## Build

1. Open in **Android Studio** (Koala+). It will provision the Gradle wrapper and
   SDK (min SDK 26, target 34, JDK 17). *(The binary `gradle/wrapper/gradle-wrapper.jar`
   is regenerated by Android Studio or `gradle wrapper`; it isn't committed.)*
2. Add the two models — see `app/src/main/assets/models/README.md`. Without them
   the app still runs and shows the chart; it just reports "model not installed".
3. Run on a device/emulator (a physical device is recommended for LLM inference).
